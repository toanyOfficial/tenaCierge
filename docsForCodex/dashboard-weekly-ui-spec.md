# 대시보드 - 주간업무 UI 용어 정리

아래 다섯 가지 레이아웃/뷰 용어를 공식 용어로 사용하며, 각 항목이 화면에서 맡는 역할과 데이터/디자인 특성을 정리한다.

## 상단 합계표
- **역할**: 요일별/일자별 전체 물량과 핵심 상태(예: 신청 완료 여부, 미신청 여부)를 한눈에 보여주는 최상단 요약 그리드.
- **데이터 원천 및 갱신**: 배치/신청 스냅샷을 집계하여 특정 시각(예: 09:00, 15:00, 16:30 등)에 갱신한다.
- **표시 요소**: 날짜, 섹터별/전체 건수, 상태별 색상(완료·미신청·기본) 등 단순 수치 중심.
- **디자인 특성**: 시간대에 따라 레이아웃 변화 없이 색상/수치만 업데이트되는 고정형 표.

## D0-dominant
- **역할**: 당일(D0) 진행 현황을 넓게 보여주는 기본 레이아웃. 섹터·건물별 진행률과 호실 단위 상태를 상세히 모니터링한다.
- **데이터 원천 및 갱신**: 실시간 혹은 짧은 주기로 스냅샷을 재요청하여 배정/청소/검수 단계별 상태를 반영한다.
- **표시 요소**: 스택형 진행 바, 최대 표시 호실 카드(플레이스홀더 포함), 단계별 색상 배지 등.
- **디자인 특성**: 화면 비중이 크며, 오버플로우 시 페이징/순환을 적용할 수 있다. 기본 모드로 진입 시 페이지나 스크롤 위치를 초기화한다.

## D0-compact
- **역할**: D0 정보를 축약해 표시하는 보조 레이아웃. 텍스트 중심의 섹터 요약 또는 축소된 목록 형태로 노출된다.
- **데이터 원천 및 갱신**: D0-dominant와 동일한 실시간/주기 스냅샷을 재사용한다.
- **표시 요소**: 섹터별 진행 수치, 최소한의 상태 배지, 필요 시 자동 페이징/순환되는 리스트.
- **디자인 특성**: 화면 비중이 작고, 컨테이너 높이에 맞춰 행 수를 계산하는 등 가용 공간에 최적화된다.

## D1-dominant
- **역할**: 익일(D1) 준비 현황을 넓게 보여주는 레이아웃. 섹터 전체 물량 대비 완료/미완료, apply 슬롯 현황을 중심으로 보여준다.
- **데이터 원천 및 갱신**: 배치/신청 데이터를 특정 시각 단위로 집계해 사용하며, role·섹터·건물 구분이 포함된다.
- **표시 요소**: 섹터별 진행률 바, 건물 구간 색상, 예약/배치 목록, 역할별 슬롯 수치와 상태 배지.
- **디자인 특성**: 15:30 이후 확장되는 기본 모드로 사용되며, 진입 시 목록/페이지 상태를 초기화한다.

## D1-compact
- **역할**: D1 정보를 압축해 표시하는 보조 레이아웃. 섹터·건물·체크아웃 시각 등 주요 구분값을 짧은 라인이나 테이블 형태로 보여준다.
- **데이터 원천 및 갱신**: D1-dominant와 동일한 집계 스냅샷을 사용하며 동일 시각에 갱신된다.
- **표시 요소**: 역할별 apply 슬롯 수치, 작업 건수, 상태 배지, 필요 시 자동 순환되는 페이징.
- **디자인 특성**: 공간 제약에 맞춰 요약 정보만 노출하며, 확대 모드로 전환 시 페이징/스크롤을 재설정한다.
